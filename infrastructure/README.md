# Infrastructure

Static SPA hosting for [audio-insights](https://audio-insights.603.nz), managed
with Terraform. Cloudflare keeps `603.nz` DNS pointed at the CloudFront
distribution, and AWS CodeBuild is the deployment runner.

## What is managed

**Static hosting** (`module.web_app`)
- Private S3 bucket with BucketOwnerEnforced ownership and no public ACLs
- CloudFront distribution fronting S3 via Origin Access Control
- SPA fallback: 403/404 -> 200 `/index.html`
- Modern TLS and AWS-managed cache policy

**TLS** (`module.acm_certificate`, `aws_acm_certificate_validation` in us-east-1)
- Per-host ACM cert for `audio-insights.603.nz`
- DNS validation records created in Cloudflare

**DNS** (`cloudflare_dns_record.acm_validation`, `cloudflare_dns_record.host`)
- ACM validation CNAME
- App `CNAME audio-insights.603.nz -> <cloudfront domain>` created after
  CloudFront is ready

**CodeBuild deployment** (`module.codebuild_role` + `module.codebuild_deploy_project`)
- Applies Terraform, builds the SPA with pnpm, syncs `dist/` to S3, and
  invalidates CloudFront
- IAM role and inline policy are generated by the shared
  `codebuild-terraform-role` module
- Webhook triggers on pushes to `master`

**SSM token placeholders**
- `/audio-insights/cloudflare-api-token` - Cloudflare API token
- `/audio-insights/ga-id` - Google Analytics ID
- `/audio-insights/spotify-client-id` - Spotify OAuth client ID

`REMOTE_STATE_BUCKET`, `APP_HOSTNAME`, Spotify callback/scopes, deployment
bucket, region, and Terraform state key are plaintext build env vars in
`buildspec.yml`; they are not secrets.

Terraform does not take a plaintext `cloudflare_api_token` variable. CodeBuild
loads `CLOUDFLARE_API_TOKEN` from SSM into the environment so the Cloudflare
provider can authenticate without storing the decrypted token in Terraform
state.

## State

Remote state is stored in S3:
`s3://jch254-terraform-remote-state/audio-insights` in `ap-southeast-4`.
## Module versions

This repo expects shared `terraform-modules` tag `1.17.0` or newer for the
modern `web-app`, `codebuild-project`, `codebuild-terraform-role`,
`acm-dns-validated-certificate`, and `ssm-parameter-placeholder` modules.

## Bootstrap

The bootstrap apply creates SSM placeholders and the hosting infrastructure.
After the first apply, populate the secret placeholders with real values.

```bash
cd infrastructure/terraform

# Authenticate with AWS first.
aws sso login  # or export AWS_* env vars

# For the first local apply, export a real Cloudflare token directly.
export CLOUDFLARE_API_TOKEN="..."

terraform init
terraform plan
terraform apply
```

Populate SSM parameters after bootstrap:

```bash
aws ssm put-parameter --region ap-southeast-4 \
  --name /audio-insights/cloudflare-api-token \
  --type SecureString --value "$CLOUDFLARE_API_TOKEN" --overwrite

aws ssm put-parameter --region ap-southeast-4 \
  --name /audio-insights/ga-id \
  --type SecureString --value "$GA_ID" --overwrite

aws ssm put-parameter --region ap-southeast-4 \
  --name /audio-insights/spotify-client-id \
  --type SecureString --value "$SPOTIFY_CLIENT_ID" --overwrite
```

The Cloudflare token needs Zone DNS Edit on `603.nz`.

## CodeBuild flow

`buildspec.yml` runs:

1. `corepack enable` + `corepack prepare pnpm@9.15.9 --activate`, then
   `pnpm install --frozen-lockfile`
2. `infrastructure/deploy-infrastructure.bash` - `terraform init` / plan /
   apply, with an IAM-first apply when needed
3. `pnpm run build` - produces `dist/`
4. `infrastructure/deploy-artifacts.bash` - `aws s3 sync dist/` and CloudFront
   invalidation

The artifact deploy step reads `DEPLOYMENT_BUCKET` and
`CLOUDFRONT_DISTRIBUTION_ID` from Terraform outputs when they are not supplied
as environment variables.
